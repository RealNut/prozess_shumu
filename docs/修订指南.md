# 修订指南 · 德语法律丛书书目（修改模式）

> 本文面向**站长 / 维护者**，介绍如何在网页端直接修订书目数据并发布上线。
> 读者视角见 `docs/读者指南.md`。本页内容**不建议对外公开链接**（含口令与令牌说明）。

## 修改模式的整体设计

站点采用**数据 / 代码分离**架构：

| 类别 | 内容 | 来源 | 谁写 |
|---|---|---|---|
| 代码 | 8 套丛书基础 JSON（de/zh/year/tags 默认值）、页面 HTML、脚本、config | 本地 `deploy.py` 部署 | 站长（经 WorkBuddy 生成） |
| 用户数据（覆盖层） | `tags.json` / `trans_overrides.json` / `year_overrides.json` / `series_order.json` | 网页端经 GitHub 令牌直写仓库 | 站长（浏览器内） |

渲染时显示值的**优先级**为：

```
本机待发布(pending)  >  已发布覆盖层(PUB)  >  基础 JSON 默认值
```

`deploy.py` 基于仓库当前 tree 重建，只替换「代码」文件，**结构性保留**覆盖层文件，因此代码部署**永远不会覆盖**网页端的修订。这是本站点最关键的容错设计。

## 第一步：解锁编辑

1. 点击页面右上角 **🔒 解锁编辑**。
2. 在弹出的对话框中输入**口令**。
3. 校验通过（浏览器本地加盐 SHA-256，明文口令不出本机）后，编辑控件出现：
   - 每行「中文译名 / 年」旁的 **✎**
   - 标签旁的 **🏷️**
   - 首页丛书卡片变为可拖拽
   - 右上角出现 **🔑 令牌** 与 **🚀 发布修改** 工具条

**门禁安全说明**：

- 口令以「加盐 SHA-256」哈希存于 `config.js` 的 `PASS_HASH`；校验在浏览器用 Web Crypto 完成，明文口令永不离开本机。
- 失败 **5 次**触发锁定，锁定时间指数退避（30s → 60s → 120s …）。
- 口令比对用**常量时间比较**，降低侧信道风险。
- 这是「个人站点轻量门禁」，**非银行级安全**。哈希与口令提示在公开代码里，真正的防线是失败锁定 + 退避；若需更强保护应改用真实后端鉴权。

## 第二步：编辑各类字段

### 中文译名 / 出版年

1. 点击该行 ✎，单元格变为输入框（窄列编辑时自动临时加宽）。
2. 输入后点 **✓** 或按 **Enter** 保存；点 **✕** 或按 **Esc** 取消。
3. 保存后改动进入「本机待发布」状态，该行左侧出现绿色标记并高亮，右上角显示「🟢 待发布 N 项」。

### 标签（行内 + 弹层 + 全局）

- **行内 🏷️ 弹层**：增（输入后回车 / 点添加）、删（×）、改（点标签名进入编辑）。
- 加标签时会给出**相似已有标签建议**（基于归一化 + 编辑距离），避免重复或错别字。
- **首页标签云**支持**全局重命名 / 删除**（二次确认），会同步影响所有含该标签的书。

### 丛书卡片顺序（首页）

解锁后，首页丛书卡片可**拖拽重排**（仅同出版社组内）。顺序为草稿，发布后生效；锁定态自动回退到已发布顺序。

## 第三步：发布

未发布前，所有改动仅存于**本机浏览器 localStorage**，仅自己可见，清浏览器数据会丢失草稿（不影响已发布内容）。

### 方式 A：GitHub 令牌直写（推荐，全员可见）

1. 点 **🔑 令牌**，填入 **GitHub fine-grained 个人访问令牌**（仅需本仓库 `contents:write`），令牌仅存本机浏览器、不上传。
2. 点 **🚀 发布修改**，网页经 Git Database API **原子提交**到覆盖层文件（单 commit，不触碰任何代码文件）。
3. 发布成功后约 20 秒自动倒计时，拉取最新覆盖层并刷新页面。

### 方式 B：复制代推（无令牌时）

未填令牌时点发布，会**复制修改 JSON 到剪贴板**，将其发到 WorkBuddy 项目对话，由助手代为推送上线。

## 改口令流程

1. 浏览器控制台执行：`await BibGate.hashFor('你的新口令')` 取得十六进制哈希。
2. 将 `config.js` 的 `PASS_HASH` 替换为新哈希（`SALT` 可一并更换，但须重新计算哈希）。
3. 重新部署（`deploy.py`）使新 `config.js` 生效。

## 注意事项

- **乐观 UI**：编辑后本机即时预览，但未发布前他人看不到。
- **清缓存风险**：清除浏览器 localStorage 会丢失「待发布」草稿，已发布内容不受影响。
- **非安全上下文**：Web Crypto 需 https 或 localhost；用 `file://` 或非安全上下文打开时无法解锁。
- **令牌权限**：fine-grained PAT 务必只授予本仓库，降低泄露影响面。
- **覆盖层文件首发布前不存在属正常**：`series_order.json` 等在未发布对应改动前仓库中本就不存在，部署健康检查会跳过，不必担心。

## 排错

| 现象 | 可能原因 | 处理 |
|---|---|---|
| 点「解锁编辑」无反应 / 提示无法校验 | 非安全上下文（如 file://） | 用 https 链接或本地 `python3 -m http.server` 以 localhost 打开 |
| 发布失败「no-token」 | 未填令牌 | 点 🔑 填入 PAT，或改用方式 B 复制代推 |
| 发布冲突 409 | 仓库刚被改动 | 脚本自动基于最新 base 重试，稍候即可 |
| 待发布计数不归零 | 发布未成功或网络中断 | 重试发布；必要时复制 JSON 代推 |
